# design/

Source files and tooling for Camp 404's UI design work. The pipeline turns the
**running app** into a navigable design system: capture clean reference
screenshots → recreate each screen in Pencil → merge into one `.pen` → implement
components from it.

## Layout

- **`brief.md`** — the token + identity source every Pencil prompt references
  (OKLCH tokens transcribed from `packages/ui/src/styles/globals.css`). **Commit.**
- **`reference/`** — clean, seeded screenshots of the live app at a mobile
  viewport, produced by `pnpm --filter @camp404/web design:capture`. These are
  the **fidelity source of truth** — commit them.
- **`prompts/`** — one Pencil prompt per screen (`<screen>.md`), each anchored to
  its `reference/<screen>.png` + the exact tokens. **Commit.**
- **`pages/`** — per-screen `.pen` files generated by Pencil. **Commit.**
- **`exports/`** — rendered previews (PNG/JPEG/PDF) from `.pen` files.
  **Gitignored** — regenerate from the `.pen` source.
- **`app.pen`** — the merged, navigable canvas (all screens on one file),
  produced by `merge-pens.mjs`. **Commit.**

## Scripts

At the **monorepo root** (`scripts/pencil/`):

- **`run.sh`** — thin wrapper around the global `@pencil.dev/cli`. Guarantees the
  pnpm global bin is on `$PATH` and **guards the machine-wide singleton**: it
  refuses to start if another `pencil` process is running (any repo), because
  concurrent runs share one global canvas and corrupt each other. Override with
  `ALLOW_CONCURRENT_PENCIL=1` only if you know the other run is idle.
- **`merge-pens.mjs`** — non-destructive JSON merge of N single-frame `.pen`
  files into one canvas (packs frames on a grid, reads real heights from the
  export PNGs, unions variables, regenerates colliding ids, validates, never
  touches sources).

`pnpm` entry points (run from `apps/web`):

- `pnpm --filter @camp404/web design:capture` — Playwright capture of all
  screens → `design/reference/*.png`. `CAPTURE_THEME=light` to flip the theme.
- `pnpm --filter @camp404/web design:cli -- <args>` — pass-through to `pencil`.
- `pnpm --filter @camp404/web design:status` — `pencil status`.

## Workflow

1. **Capture** clean references: `pnpm --filter @camp404/web design:capture`.
   Review `design/reference/*.png` — populated, no banners/overlays, mobile width.
2. **Recreate** each screen, one at a time (serial — singleton canvas):
   ```bash
   scripts/pencil/run.sh \
     --out design/pages/<screen>.pen \
     --workspace design/reference \
     --prompt "$(cat design/prompts/<screen>.md)" \
     --export design/exports/<screen>.png
   ```
   Then eyeball `design/exports/<screen>.png` against `design/reference/<screen>.png`.
3. **Merge** into one canvas:
   ```bash
   node scripts/pencil/merge-pens.mjs --out design/app.pen design/pages/
   ```
4. **Implement** components in `packages/ui/src/components/` from the exports + tokens.

## Prerequisites

```bash
npm i -g @pencil.dev/cli      # global tool, not a repo dependency
pencil login                  # stores ~/.pencil/session-cli.json
scripts/pencil/run.sh status  # expect: ● Active
```

Pin the CLI version (Pencil is young, no public changelog — gate upgrades behind
a smoke test). For CI/unattended runs use `PENCIL_CLI_KEY` instead of `pencil login`.

> **Durable source of truth = the running app + committed reference PNGs +
> prompts + brief — NOT the `.pen`.** If a `.pen` breaks, regenerate from the prompt.

See [`../docs/design-tooling.md`](../docs/design-tooling.md) for the pencil.dev
workflow rationale and [`../docs/design-system.md`](../docs/design-system.md) for
the `@camp404/ui` tokens and components every mockup should target.
